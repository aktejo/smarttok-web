# SmartTok (web)

An endless feed of curated content — Wikipedia, Dogs, Cats, arXiv, PubMed,
NASA APOD, Poetry, Project Gutenberg, and CORE (open-access research) —
pulled live from public APIs. Plain HTML/CSS/JS, no build step, no framework.

> CORE works keyless but its anonymous quota is tiny. A free registered key
> (https://core.ac.uk/services/api) lifts it: `vercel env add CORE_API_KEY`,
> then redeploy — the proxy injects it server-side, so it never reaches
> browsers or the repo.

## Running locally

The full app — including the arXiv/PubMed proxy — runs locally with:

```bash
vercel dev --listen 3000
```

Then open `http://localhost:3000`. This serves the static files *and* runs
`api/proxy.js` as a local serverless function, so every source works.

If you don't need arXiv/PubMed, any static file server also works:

```bash
python3 -m http.server 8000
# or: npx serve .
```

**Wikipedia, Dogs, Cats, and NASA** work immediately this way — their APIs
send the right CORS headers for direct browser access.

**arXiv and PubMed will show "couldn't load this one" cards** under a plain
static server, because neither of those APIs sends CORS headers, so a
browser can't `fetch()` them directly no matter how the adapter code is
written — they need the proxy.

## Verifying the sources

Open **`/verify.html`** (locally under `vercel dev`, or on the deployed
site). It runs a live end-to-end check of every registered source: real
fetch, NormalizedContent contract validation, batch-unique IDs, an
image-load probe, and a sample card rendered through the actual feed
pipeline — plus allowlist/param checks on the proxy. Each source gets a
PASS/FAIL row; results are also exposed on `window.__verifyResults` for
scripted use. Add a source to `adapters/index.js` and it's covered
automatically.

## Why a proxy?

`export.arxiv.org` and `eutils.ncbi.nlm.nih.gov` are both free, keyless,
public APIs — but neither returns an `Access-Control-Allow-Origin` header,
which means browsers block client-side `fetch()` calls to them regardless of
origin. This isn't a bug in the adapter; it's a deliberate (or at least
unaddressed) choice on those services' end.

The fix is `api/proxy.js`: a small serverless function that makes the
request server-side (servers aren't subject to CORS) and re-serves the
response with its own CORS header attached. Any adapter can use it by
wrapping its target URL with `proxiedUrl(...)` from `js/proxy-config.js`
instead of calling the upstream API directly. It's a generic relay, not
arXiv/PubMed-specific — allowlisted hosts live in `api/proxy.js`.

## Deploying the proxy (Vercel)

1. Install the Vercel CLI if you don't have it: `npm i -g vercel`
2. From this project folder: `vercel deploy`
3. Follow the prompts (first deploy will ask you to link/create a project)

That's it — `vercel.json` is already configured, and `api/proxy.js` will
deploy automatically as a serverless function at `/api/proxy`.

**If you deploy the static site and the proxy together** (the default —
just `vercel deploy` from this folder), no further config is needed. The
adapters call `/api/proxy` on the same origin.

**If you deploy them separately** (e.g. the static site on GitHub Pages or
Netlify, with only the proxy on Vercel), open `js/proxy-config.js` and set
`PROXY_BASE_URL` to your deployed proxy's full URL, e.g.:

```js
const PROXY_BASE_URL = "https://your-proxy.vercel.app";
```

## AI summaries (on-device, free)

Research-paper cards (arXiv, PubMed, CORE) can show a one-line plain-English
restatement of the abstract, inserted between the title and the body. It's generated by the
browser's **built-in Summarizer API** (Gemini Nano in Chrome 138+ on
desktop): no API key, no server, no cost — the model runs on-device and
card text never leaves the browser.

- In supported browsers: **Settings → AI summaries → Download model &
  enable** (Chrome downloads the model once; it works offline after that).
- In unsupported browsers the feature simply doesn't appear.
- Summaries are cached per card in localStorage, so nothing is summarized
  twice.
- A source opts in by setting `simplify: true` on its adapter — see
  `js/summarizer.js` for the mechanics.

## Adding a new source

1. Create `adapters/yoursource.js` — see `adapters/PROTOCOL.js` for the
   contract every adapter follows.
2. If the upstream API sends CORS headers, call it directly with `fetch()`.
   If not, route through the proxy:
   - add the API's hostname to `ALLOWED_HOSTS` in `api/proxy.js`
   - call `fetch(proxiedUrl(targetUrl))` instead of `fetch(targetUrl)`
3. Add one line to `adapters/index.js` to register it, and one `<script>`
   tag in `index.html` to load the file.

Nothing else — Settings, the Mixer, and the feed all pick up new sources
automatically from the registry.
